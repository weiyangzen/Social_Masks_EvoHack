# Social Mask Skills

`MasksSkills/` 存放 A2A 社交面具的具体 skill 定义文件。

这是**源蓝图层**，用于维护原始关系场景设计与业务语言。

如果要：

- 安装到通用 skill 系统
- 导出到公开环境
- 发布到 EvoMap

请优先使用仓库根目录下的 `DialogueProfilesSkills/` 导出层，而不是直接拿 `MasksSkills/` 原文外发。

## 目录规则
- 每个关系目录使用固定 ASCII 编号路径，例如 `r01_family/`、`r02_colleague/`。
- 每个方面目录也使用固定 ASCII 编号路径，例如 `a01_emotional_support/`。
- 每个具体面具 skill 文件使用 `iNN.md` 命名。

## 完成判定
- 蓝图中的一个勾选项只有在对应 skill 文件已经存在时才能标记为 `[x]`。
- skill 文件必须包含下面的统一标题，缺任何一项都不算完成。

## 统一模板
```md
# <面具名称>

## Skill Card
- Skill ID:
- 关系:
- 方面:
- 蓝图锚点:
- 文件路径:

## 面具定位

## 解决的问题

## 典型触发信号

## 输入槽位

## 输出要求

## 执行步骤

## 话术与策略

## 边界与禁区

## 失败风险

## 成功判定

## 示例开场
```

## 写作要求
- 面向 A2A 社交面具系统，内容必须具体、可执行，不能只写概念口号。
- `执行步骤` 至少写 4 步，强调先判断场景，再组织表达，再收口确认。
- `边界与禁区` 必须明确不能代替真人做的决定、不能诱导违法、不能操控或诈骗。
- `示例开场` 至少给 2 条简短示例。
